     Deploy Zubro Food Cost App to Google Cloud

     Context

     The app is a FastAPI + React monolith using SQLite. The goal is to run it
     on GCP with persistent data and secret management. No Dockerfile exists yet
      and the backend does not serve the frontend build — both must be added
     before deployment.

     ---
     Recommended approach: Cloud Run + Cloud Storage volume mount

     Cloud Run is serverless (pay only when used), scales to zero, and v2
     supports mounting a GCS bucket as a filesystem volume, which keeps SQLite
     working without migrating to another database engine.

     Alternative (simpler ops, always-on): Compute Engine e2-small VM +
     persistent disk (~$12/mo). Swap in if you want a traditional VM instead.

     ---
     What needs to be built / changed

     1. Serve the frontend from FastAPI (backend change)

     Add StaticFiles mount so the production build of the React app is served by
      the FastAPI server.

     File: backend/app/main.py
     from fastapi.staticfiles import StaticFiles
     # After all API routers are registered, at the very bottom:
     app.mount("/", StaticFiles(directory="../frontend/dist", html=True),
     name="static")
     The existing /api prefix on all routes means there is no path conflict.

     2. Multi-stage Dockerfile (new file at repo root)

     Stage 1 — build React frontend with Node, Stage 2 — Python image with
     compiled frontend copied in.

     # Stage 1: build frontend
     FROM node:20-alpine AS frontend-build
     WORKDIR /app/frontend
     COPY frontend/package*.json ./
     RUN npm ci
     COPY frontend/ .
     RUN npm run build

     # Stage 2: Python backend
     FROM python:3.11-slim
     WORKDIR /app/backend
     COPY backend/requirements.txt .
     RUN pip install --no-cache-dir -r requirements.txt
     COPY backend/ .
     COPY --from=frontend-build /app/frontend/dist ../frontend/dist
     EXPOSE 8000
     CMD ["python", "run.py"]

     3. Production run.py tweak (remove reload flag)

     backend/run.py currently passes reload=True unconditionally. Add an
     env-variable guard so reload is disabled in production:
     import os
     reload = os.getenv("ENV", "development") == "development"
     uvicorn.run("app.main:app", host="0.0.0.0", port=8000, reload=reload)

     4. CORS: allow the production domain

     backend/app/main.py currently only allows localhost:5173 and
     localhost:3000. After deploying to Cloud Run you'll get a *.run.app URL.
     Either:
     - Set allow_origins=["*"] temporarily, or
     - Add the Cloud Run URL via an ALLOWED_ORIGINS env var.

     ---
     GCP deployment steps

     Prerequisites (one-time)

     # Install gcloud CLI and authenticate
     gcloud auth login
     gcloud config set project YOUR_PROJECT_ID
     gcloud services enable run.googleapis.com \
         artifactregistry.googleapis.com \
         storage.googleapis.com \
         secretmanager.googleapis.com

     5. Create a GCS bucket for SQLite persistence

     gcloud storage buckets create gs://zubro-food-cost-db
     --location=europe-west1
     Upload your current database to seed it:
     gcloud storage cp zubro_food_cost.db
     gs://zubro-food-cost-db/zubro_food_cost.db

     6. Store secrets in Secret Manager

     echo -n "YOUR_ANTHROPIC_KEY" | gcloud secrets create ANTHROPIC_API_KEY
     --data-file=-
     echo -n "YOUR_GOOGLE_KEY"    | gcloud secrets create GOOGLE_API_KEY
     --data-file=-

     7. Build and push the container

     gcloud artifacts repositories create zubro \
         --repository-format=docker --location=europe-west1

     docker build -t europe-west1-docker.pkg.dev/YOUR_PROJECT/zubro/app:latest .
     docker push europe-west1-docker.pkg.dev/YOUR_PROJECT/zubro/app:latest
     Or use Cloud Build to build without Docker installed locally:
     gcloud builds submit --tag
     europe-west1-docker.pkg.dev/YOUR_PROJECT/zubro/app:latest

     8. Deploy to Cloud Run v2 with GCS volume mount

     gcloud run deploy zubro-food-cost \
         --image europe-west1-docker.pkg.dev/YOUR_PROJECT/zubro/app:latest \
         --region europe-west1 \
         --platform managed \
         --port 8000 \
         --memory 512Mi \
         --cpu 1 \
         --min-instances 0 \
         --max-instances 2 \
         --set-env-vars ENV=production \
         --set-secrets ANTHROPIC_API_KEY=ANTHROPIC_API_KEY:latest,GOOGLE_API_KEY
     =GOOGLE_API_KEY:latest \
         --add-volume name=db-vol,type=cloud-storage,bucket=zubro-food-cost-db \
         --add-volume-mount volume=db-vol,mount-path=/db \
         --set-env-vars DB_PATH=/db/zubro_food_cost.db \
         --allow-unauthenticated

     9. Update DB_PATH in config

     backend/app/config.py currently hard-codes the path relative to the source
     tree. Make it read from env:
     db_path: str = Field(
         default=str(Path(__file__).parent.parent.parent / "zubro_food_cost.db")
     )
     The --set-env-vars DB_PATH=/db/zubro_food_cost.db flag above overrides the
     default at runtime — Pydantic Settings reads env vars automatically, so no
     other change is needed.

     ---
     Verification

     1. Local Docker test: docker build -t zubro . && docker run -p 8000:8000 -e
      DB_PATH=/tmp/test.db zubro — open http://localhost:8000, confirm frontend
     loads and API calls work.
     2. After Cloud Run deploy: open the printed *.run.app URL, log in, check
     the dashboard loads and data persists across requests.
     3. Re-deploy with a code change → confirm DB data survives the redeploy
     (GCS-backed volume).

     ---
     Estimated cost (europe-west1)

     - Cloud Run (< 2M requests/month): effectively free tier
     - Cloud Storage (< 1 GB bucket): < $0.03/month
     - Secret Manager: free tier
     - Artifact Registry: free for first 0.5 GB

     Total: ~$0–2/month for typical internal-tool usage.

❯ what is the YOR_PROJECT variable?                                             

● It's your Google Cloud Project ID — a unique string you choose (or that GCP
  auto-generates) when you create a GCP project. It's not the same as the
  project name.

  To find it:
  - Go to https://console.cloud.google.com → the project selector dropdown at
  the top shows both the name and the ID (e.g. my-project-123456)
  - Or run: gcloud projects list

  If you haven't created a GCP project yet:
  gcloud projects create zubro-food-cost --name="Zubro Food Cost"
  gcloud config set project zubro-food-cost

  Then replace every YOUR_PROJECT in the deployment commands with
  zubro-food-cost (or whatever ID you chose). The full Artifact Registry image
  tag would look like:

  europe-west1-docker.pkg.dev/zubro-food-cost/zubro/app:latest
